2. How Data Is Processed

1

Local Recording

The extension records foreground tab hostname and time spent at short intervals.

2

Block Checks

Block checks run locally first (URL + keywords). Optional in-page keyword scanning reads visible text only; it never transmits page bodies to the server‚Äîonly the fact a keyword matched.

3

Sync Process

Data buffers locally and syncs periodically to the backend API over HTTPS.

4

Report Generation

Reports (email summaries) are generated using your already synced, structured data‚Äînot raw browsing history.

3. Optional Content Keyword Scanning

You can disable in-page keyword scanning in the Guard tab. When disabled:

  • Only URL + domain name matching is used.
  • No page text is read or analyzed.
